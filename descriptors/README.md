# Descriptor Artifacts

This directory contains stored xcube `DataDescriptor` artifacts for CCI datasets from the Open Data Portal.

Descriptors are generated by the metadata builder and should be treated as
versioned cache artifacts for the standard `xcube-cci` access mode. They are
not the only source of truth, but they allow `describe()`-style APIs to avoid
expensive live ODP calls for common cases.

Descriptor files use the structure defined by xcube. This repository should not
invent a parallel descriptor schema.

Descriptors are stored for the default `normalize_data=True` behavior. Separate
raw descriptor variants are intentionally out of scope.

Suggested layout:

```text
descriptors/
  esa-cci/
    <safe-data-id>.json
```

Only ODP descriptors are cached initially. Zarr and Kerchunk descriptors are
cheap enough to compute live and can be added later if a concrete need appears.
The descriptor payload itself carries the data type, so separate data-type
subdirectories are not required.

The descriptors in this directory apply to the `xcube-cci` version recorded in
`../build_info.json`. Older `xcube-cci` versions are supported by older
`xcube-cci-registry` releases, not by parallel version directories.

If descriptor content is unchanged across releases, Git history already avoids
conceptual duplication and keeps old snapshots available through tags.
